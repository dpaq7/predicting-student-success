# Predicting Student Success

Machine learning capstone project for **CIND820: Big Data Analytics Project** at Toronto Metropolitan University.

This project investigates whether modern educational analytics methods can improve student performance prediction and support earlier, more explainable academic intervention. It extends the foundational student performance prediction work by Cortez and Silva (2008) using the UCI Student Performance Dataset.

## Project Context

Educational institutions often identify struggling students after poor grades have already accumulated. At that point, the intervention window is smaller and academic confidence may already be damaged. This project studies whether machine learning can identify at-risk students earlier, explain the strongest risk factors, and suggest differentiated intervention strategies.

The project is framed as an academic analytics workflow rather than a production student-risk system. The emphasis is on reproducible analysis, model comparison, interpretation, and ethical discussion.

## Research Questions

1. **Algorithm benchmarking:** How much do modern machine learning methods improve on the Cortez and Silva benchmarks of approximately 93% classification accuracy and RMSE around 2.0 for grade regression?
2. **Early warning timing:** How early can at-risk students be identified while still reaching an actionable accuracy threshold?
3. **Explainability:** Which features explain the strongest risk signals in terms educators can interpret?
4. **Student archetypes:** Do unsupervised student clusters suggest different intervention strategies for different student profiles?

## Dataset

The project uses the public UCI Student Performance Dataset:

- `student-mat.csv`: 395 mathematics students
- `student-por.csv`: 649 Portuguese-language students
- Combined dataset: 1,044 rows
- Original features: 33
- Classification target: `pass_fail`, defined as `G3 >= 10`
- Regression target: final grade `G3` on a 0-20 scale

The dataset includes demographic, social, family, school, behavioral, support, absence, prior-failure, and grade-period features.

Raw data files are stored in:

- `data/01_raw/student-mat.csv`
- `data/01_raw/student-por.csv`

The raw files are treated as immutable. Derived outputs are regenerated by the analysis pipeline.

## Methodology

The project uses temporal feature sets to avoid overstating early-warning performance:

- **T0 no current grades:** demographic, school, family, social, support, study, behavior, and prior-failure features.
- **T1 after first grade:** T0 plus current absences and first-period grade `G1`.
- **T2 after second grade:** T1 plus second-period grade `G2`.

This design separates true early-warning prediction from later prediction that has higher accuracy but leaves less time for intervention.

### Models

Classification models:

- Logistic Regression
- Random Forest
- Histogram Gradient Boosting

Regression models:

- Linear Regression
- Ridge Regression
- Random Forest Regressor
- Histogram Gradient Boosting Regressor

Evaluation uses cross-validation and reports:

- Classification: accuracy, balanced accuracy, precision, recall, F1, ROC-AUC
- Regression: RMSE, MAE, R2
- Efficiency: fit time
- Stability: cross-validated performance

### Explainability and Clustering

The project uses permutation importance as a package-light, model-agnostic explanation method. The implementation is structured so SHAP can be added later if package compatibility is available.

Student archetypes are explored with k-means clustering on T0 features. Clusters are profiled by pass rate, final grade, failures, absences, study time, and social/behavioral features.

## Key Results

Latest generated results:

- Best no-current-grade early warning model: **Random Forest**, 0.788 accuracy, 0.794 ROC-AUC.
- First actionable timing point: **T1 after first-period grade**, Hist Gradient Boosting, 0.875 accuracy.
- Best full-feature classifier: **Hist Gradient Boosting on T2**, 0.916 accuracy, 0.969 ROC-AUC.
- Best full-feature regressor: **Hist Gradient Boosting on T2**, RMSE 1.438, improving on the Cortez and Silva RMSE around 2.0 benchmark.
- Cluster analysis identified one clearly high-risk archetype: 108 students, 0.417 pass rate, 7.815 mean final grade, and 1.806 mean prior failures.

Interpretation:

- Enrollment-time prediction is close to the 80% action threshold but does not cross it.
- The first-period grade is the most practical intervention trigger in this dataset.
- Interim grades dominate prediction once available, but absences and prior failures remain useful intervention signals.
- Cluster results are useful for exploratory intervention planning but should not be treated as definitive segmentation because cluster separation is weak.

## Reproducing the Analysis

### Prerequisites

- Python 3.10 or higher
- `uv` package manager

The local environment used for this run is Python 3.13.5. The project uses scikit-learn-compatible models to avoid optional dependency issues with gradient boosting and SHAP libraries.

### Setup

```bash
git clone https://github.com/dpaq7/predicting-student-success.git
cd predicting-student-success
make init_env
source .venv/bin/activate
make install_data_libs
```

### Run the Full Pipeline

```bash
python -m src.pipelines.training_pipeline.run_analysis
```

- `data/03_primary/student_performance_combined.csv`

## Limitations

- The dataset represents two Portuguese secondary schools, so generalizability to other contexts is limited.
- The dataset is small by modern machine learning standards.
- Some behavioral and social variables are self-reported.
- Current-year absences must be handled carefully when making early-warning claims.
- The explainability layer currently uses permutation importance rather than SHAP.
- K-means cluster separation is weak, so archetypes are exploratory.

## Ethical Considerations

Student risk prediction should support educators, not replace human judgment. Risk labels can stigmatize students if used punitively or without context. Demographic and family-background variables may encode structural inequities, so any real deployment would require privacy review, bias testing, transparent communication, and intervention-focused use.
